File: packages/generator/src/cli/theme-manager.ts

The ThemeManager class handles theme-related operations, such as parsing user input, generating CSS, and retrieving theme-specific assets. It integrates directly with the CSS system to create visually consistent output for each theme.

Highlights

  • generateThemeCSS: Combines base styles with theme-specific CSS files to create a cohesive stylesheet.
  • getThemeIcons: Retrieves icons specific to the selected theme for consistent visual representation.
  • parseThemeArg: Parses command-line arguments to determine the selected theme or handle "all" themes.

Code

generateThemeCSS(theme: AdventureTheme, outputDir: string): void {
  const cssFiles = [
    path.join(this.themesDir, 'base.css'),
    path.join(this.themesDir, 'homepage.css'), 
    path.join(this.themesDir, 'animations.css'),
    path.join(this.themesDir, `${theme}.css`)
  ];
  
  let combinedCSS = '';
  
  cssFiles.forEach(cssFile => {
    if (fs.existsSync(cssFile)) {
      combinedCSS += fs.readFileSync(cssFile, 'utf8') + '\\n\\n';
    }
  });
  
  const cssPath = path.join(outputDir, 'assets', 'theme.css');
  fs.writeFileSync(cssPath, combinedCSS);
}
  • The generateThemeCSS method dynamically assembles a complete stylesheet for the selected theme.
  • Base styles ensure consistency, while theme-specific files add unique visual elements.
  • File existence checks make the method robust and prevent runtime errors.
  • This approach modularizes CSS generation, making it easier to maintain and expand.
  • The method demonstrates how to integrate file system operations with runtime logic.
