packages/generator/src/cli/template-engine.ts: Template Management

The TemplateEngine is responsible for loading, caching, and rendering dynamic HTML templates into static files. Through efficient file I/O operations and placeholder substitution, this module ensures reusable templates support enriched site generation.

Highlights

  • loadTemplate: Caches templates for reuse, optimizing file reads and preventing redundant processing. This ensures performance scalability when dealing with multiple templates.
  • replacePlaceholders: Dynamically replaces placeholders (e.g., {{PLACEHOLDER}}) in the HTML template with actual values passed as variables.
  • renderPage: Combines a base layout and content template, allowing modular generation of dynamic pages with hierarchical templates.

Code

packages/generator/src/cli/template-engine.ts

private loadTemplate(templateName: string): string {
  if (this.templateCache.has(templateName)) {
    return this.templateCache.get(templateName)!;
  }

  const templatePath = path.join(this.templatesDir, templateName);
  
  try {
    const content = fs.readFileSync(templatePath, 'utf-8');
    this.templateCache.set(templateName, content);
    return content;
  } catch (error) {
    throw new Error(`Failed to load template ${templateName}: ${error}`);
  }
}
  • Caches templates using Map, reducing repeated reads for efficiency.
  • Leverages fs.readFileSync for synchronous file access, ensuring immediate availability during the rendering pipeline.
  • Falls back to error reporting in case of missing or inaccessible templates, aiding debugging.
private replacePlaceholders(template: string, variables: TemplateVariables): string {
  let result = template;
  
  for (const [key, value] of Object.entries(variables)) {
    const placeholder = `{{${key}}}`;
    result = result.replace(new RegExp(placeholder, 'g'), value || '');
  }
  
  return result;
}
  • Uses RegExp to globally match placeholders in the form of {{KEY}} for precise substitution.
  • Iteratively applies values, enabling seamless template customization for different contexts.
  • Ensures unused placeholders are rendered as empty strings, keeping templates predictive.
renderPage(contentTemplate: string, variables: TemplateVariables): string {
  const baseTemplate = this.loadTemplate('base-template.html');
  const contentHtml = this.loadTemplate(contentTemplate);

  const renderedContent = this.replacePlaceholders(contentHtml, variables);
  const pageVariables = { ...variables, CONTENT: renderedContent };

  return this.replacePlaceholders(baseTemplate, pageVariables);
}
  • Renders a complete page by combining a base-template with specific contentTemplate.
  • Reuses replacePlaceholders for consistent substitution patterns.
  • Enforces separation of concerns by allowing the CONTENT variable to be nested within the base structure.

packages/generator/src/cli/asset-manager.ts: Asset Management

The AssetManager handles effective organization and copying of assets like CSS, images, and JavaScript files. It splits assets between shared resources and theme-specific files, ensuring streamlined multi-theme compatibility.

Highlights

  • copyGlobalAssets: Consolidates global shared resources (e.g., GitHub icons) and makes them accessible across multiple themes.
  • copyQuestNavigator: Copies theme-specific files for quest navigation.
  • copyImages: Manages image assets by separating shared files and theme configurations.

Code

packages/generator/src/cli/asset-manager.ts

copyGlobalAssets(outputDir: string): void {
  const globalAssetsDir = path.join(outputDir, 'assets', 'shared');
  fs.mkdirSync(globalAssetsDir, { recursive: true });

  const globalImages = ['github-mark.svg', 'github-mark-white.svg'];
  globalImages.forEach(file => {
    const sourcePath = path.join(this.sourceDir, 'assets', 'shared', file);
    fs.copyFileSync(sourcePath, path.join(globalAssetsDir, file));
  });
}
  • Copies global assets like GitHub icons into a shared directory, optimizing reuse across themes.
  • Scopes global resources to prevent redundancy.
  • Uses fs.copyFileSync for synchronous processing, ensuring file consistency.
copyQuestNavigator(outputDir: string): void {
  const templatesDir = path.join(this.sourceDir, 'templates');
  const assetsDir = path.join(outputDir, 'assets');
  fs.mkdirSync(assetsDir, { recursive: true });

  const cssSource = path.join(templatesDir, 'quest-navigator.css');
  if (fs.existsSync(cssSource)) {
    fs.copyFileSync(cssSource, path.join(assetsDir, 'quest-navigator.css'));
  }
}
  • Transfers navigation-related files to theme-specific directories.
  • Ensures independent styling per theme without risk of conflicts.
  • Provides critical files (e.g., quest-navigator.css) for consistent user interface.
copyImages(outputDir: string, isMultiTheme: boolean): void {
  const targetImagesDir = path.join(outputDir, 'assets', 'images');
  fs.mkdirSync(targetImagesDir, { recursive: true });

  const files = fs.readdirSync(path.join(this.sourceDir, 'assets', 'images'));
  files.forEach(file => {
    const source = path.join(this.sourceDir, 'assets', 'images', file);
    const target = path.join(targetImagesDir, file);
    fs.copyFileSync(source, target);
  });
}
  • Organizes images into a centralized directory for site consistency.
  • Differentiates between global and theme-specific files for flexibility.
  • Simplifies directory creation using recursive options, supporting nested structures.
