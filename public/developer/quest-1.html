packages/mcp/src/server.ts: MCP Server Initialization and Tool Request Handling

This file is responsible for initializing the MCP server, handling incoming requests, and orchestrating tool execution. It defines the RepoAdventureServer class, which establishes the foundation for the dynamic interaction between tools and the user. We will focus on the setupHandlers and run methods, as well as the main entry point function.

Highlights

  • setupHandlers: Establishes handlers for listing available tools and executing tools based on user requests. It uses schemas for validation and ensures dynamic tool functionality is handled seamlessly.
  • run: Prepares and executes the server's core operations, such as connecting the transport layer and warming up resource-heavy utilities (e.g., repo analysis) to enhance responsiveness.
  • main: Acts as the program's entry point, managing initialization, graceful shutdowns, and error handling for server processes.

private setupHandlers() {
  // Dynamic tool listing
  this.server.setRequestHandler(ListToolsRequestSchema, async () => {
    const toolList = Object.entries(tools).map(([name, tool]) => ({
      name,
      description: tool.description,
      inputSchema: zodToJsonSchema(tool.schema, { 
        target: 'jsonSchema7',
        $refStrategy: 'none'
      })
    }));

    return { tools: toolList };
  });

  // Dynamic tool execution
  this.server.setRequestHandler(CallToolRequestSchema, async (request) => {
    try {
      const { name, arguments: args } = request.params;

      if (!(name in tools)) {
        throw new McpError(ErrorCode.MethodNotFound, `Unknown tool: ${name}`);
      }

      const tool = tools[name as keyof typeof tools];
      
      // Validate arguments using the tool's Zod schema
      const validationResult = tool.schema.safeParse(args);
      if (!validationResult.success) {
        const errorMessages = validationResult.error.issues.map((err) => 
          `${err.path.join('.')}: ${err.message}`
        ).join(', ');
        throw new McpError(ErrorCode.InvalidParams, `Invalid parameters: ${errorMessages}`);
      }

      // Execute the tool handler with validated arguments
      return await tool.handler(validationResult.data as any);

    } catch (error) {
      if (error instanceof McpError) {
        throw error;
      }
      throw new McpError(
        ErrorCode.InternalError,
        `Tool execution failed: ${error instanceof Error ? error.message : String(error)}`
      );
    }
  });
}
  • The ListToolsRequestSchema handler dynamically constructs a list of tools, converting validation schemas into JSON.
  • The CallToolRequestSchema handler validates and executes tools by leveraging their individual schema and handler configurations.
  • Error handling is robust, with consistent MCP error types used for standardization.
async run() {
  const transport = new StdioServerTransport();
  await this.server.connect(transport);
  console.error('Repo Adventure MCP server running on stdio');
  
  // Pre-generate repomix content for the current working directory to warm up the cache
  // This happens in the background while waiting for user commands
  const projectPath = process.cwd();
  console.error(`Pre-generating repomix content for project at ${projectPath}...`);
  repoAnalyzer.preGenerate(projectPath);
}
  • StdioServerTransport connects the server for standard input/output communication.
  • Pre-generating content with repoAnalyzer.preGenerate ensures smoother runtime experiences.
  • This asynchronous setup emphasizes efficient initialization without blocking tool availability.
async function main() {
  try {
    const server = new RepoAdventureServer();
    
    // Handle graceful shutdown for both signals
    ['SIGINT', 'SIGTERM'].forEach(sig => 
      process.on(sig as NodeJS.Signals, gracefulShutdown)
    );
    
    // Handle unhandled promise rejections - log but don't shutdown during normal operation
    process.on('unhandledRejection', (reason) => {
      console.error('Unhandled promise rejection:', reason);
      console.error('MCP server continuing to run. Please report this error.');
      // Don't call gracefulShutdown() here as it may be a recoverable error
    });
    
    await server.run();
  } catch (error) {
    console.error('Fatal error starting MCP server:', error);
    process.exit(1);
  }
}
  • The main function initializes the server and installs handlers for orderly shutdowns.
  • It also captures unhandled promise rejections, ensuring operational resilience during runtime.
