🚀 Starship Cartographer: The Repo Reconnaissance

🚀 Adventure Awaits!

You are a navigator aboard the Starship Cartographer, a compact exploration vessel tasked with mapping a newly discovered repository nebula and turning its code constellations into actionable, story-driven quests for the crew. The ship uses a Repomix-powered scanner to analyze code terrain, an LLM comms array to craft themed narratives and quest briefs, and an MCP protocol bridge to present four core tools that crew members invoke from mission control. Your mission: transform raw project artifacts into an engaging sequence of explorations that teach, test, and reveal the repository's inner workings through targeted story chapters using server, analyzer, LLM, and theme systems.

🗺️ Your Quest Awaits – Choose your path wisely, brave adventurer!

🌌 Choose a Quest

🚀 Quest 1: Command Deck — MCP Interface and Server Handlers

You explore the command deck by examining how the MCP server establishes protocol handlers, manages stdio transport, and pre-generates repository context to expose the four main tools for crew interaction, focusing on MCP handlers and startup flow in server.ts.

🚀 Quest 2: Sensor Array — Repomix Analysis and Subprocess Capture

You explore the sensor array that runs Repomix scans and captures CLI output to produce targeted or full project context, focusing on subprocess execution, caching, and project path validation in repo-analyzer.ts.

🚀 Quest 3: Story Engine — LLM-Driven Narrative and Quest Creation

You explore the story engine that crafts themed narratives and quest payloads with prompt construction, timeout handling, and LLM response validation using the LLM client and story-generator to convert project metadata into adventures.

🚀 Quest 4: Mission Control — Adventure State and Quest Orchestration

You explore mission control where the adventure manager tracks progress, initializes adventures with project context, merges config-driven file lists, validates choices, and executes quest generation flows tied to story state and caching.

🚀 Quest 5: Crew Tools — MCP Tool Definitions and Handlers

You explore the crew tools which present the user-facing MCP commands that start adventures, choose themes, explore individual quests, and view progress, connecting UI actions to adventure back-end behavior.

🚀 Quest 6: Charting Parameters — Themes, Validation, and Adventure Config

You explore the ship's navigation charts by reviewing theme definitions, theme validation, input sanitization, and the adventure configuration loader that formats quest file lists for LLM prompts and file path extraction.

🚀 Quest 7: Lifeline — Configuration and Timeout Policies

You explore the lifeline systems that govern request timeouts, caching TTLs, and environment-driven settings which protect long-running LLM and Repomix operations during story generation missions.
