File: packages/generator/src/cli/template-engine.ts

The TemplateEngine class is the heart of the Scroll of HTML Weaving, enabling the dynamic rendering of HTML templates. It uses a caching mechanism to load templates efficiently and replaces placeholders with actual content. This ensures that adventures are rendered with both speed and precision, making it a cornerstone of the HTML generation process.

Highlights

  • loadTemplate: Loads and caches HTML templates from the filesystem to optimize performance.
  • replacePlaceholders: Replaces placeholders in templates with provided variables using a flexible substitution mechanism.
  • renderPage: Combines a base template with a content template to generate a complete HTML page.

Code

private loadTemplate(templateName: string): string {
  if (this.templateCache.has(templateName)) {
    return this.templateCache.get(templateName)!;
  }

  const templatePath = path.join(this.templatesDir, templateName);
  
  try {
    const content = fs.readFileSync(templatePath, 'utf-8');
    this.templateCache.set(templateName, content);
    return content;
  } catch (error) {
    throw new Error(`Failed to load template ${templateName}: ${error}`);
  }
}
  • Implements a caching mechanism to avoid redundant file reads, enhancing performance.
  • Ensures templates are loaded from a centralized directory, maintaining consistency.
  • Handles errors gracefully, providing clear feedback when templates fail to load.
private replacePlaceholders(template: string, variables: TemplateVariables): string {
  let result = template;
  
  for (const [key, value] of Object.entries(variables)) {
    const placeholder = `{{${key}}}`;
    result = result.replace(new RegExp(placeholder, 'g'), value || '');
  }
  
  return result;
}
  • Uses regular expressions to dynamically replace placeholders with variable values.
  • Supports multiple replacements within a single template, making it versatile.
  • Ensures that missing variables do not break the rendering process by defaulting to empty strings.
renderPage(contentTemplate: string, variables: TemplateVariables): string {
  const baseTemplate = this.loadTemplate('base-template.html');
  const contentHtml = this.loadTemplate(contentTemplate);
  
  const renderedContent = this.replacePlaceholders(contentHtml, variables);
  
  const pageVariables = {
    ...variables,
    CONTENT: renderedContent
  };
  
  return this.replacePlaceholders(baseTemplate, pageVariables);
}
  • Combines a base template with a content-specific template for modular HTML generation.
  • Leverages replacePlaceholders to dynamically populate both templates with variables.
  • Ensures reusability by decoupling base layout from content-specific logic.

File: packages/generator/src/cli/asset-manager.ts

The AssetManager class is a crucial ally in managing the enchanted artifacts of the Scroll. It handles the copying and organization of assets, ensuring that images, CSS, and other resources are properly placed for both single and multi-theme adventures.

Highlights

  • copyImages: Copies theme-specific and shared images to the output directory.
  • copyQuestNavigator: Copies navigation assets for single-theme adventures.
  • copyGlobalAssets: Manages shared assets for multi-theme configurations.

Code

copyImages(outputDir: string, isMultiTheme: boolean = false): void {
  const sourceImagesDir = path.join(this.sourceDir, 'assets', 'images');
  const targetImagesDir = path.join(outputDir, 'assets', 'images');

  try {
    if (fs.existsSync(sourceImagesDir)) {
      fs.mkdirSync(targetImagesDir, { recursive: true });
      const imageFiles = fs.readdirSync(sourceImagesDir);
      imageFiles.forEach(file => {
        const sourcePath = path.join(sourceImagesDir, file);
        const targetPath = path.join(targetImagesDir, file);
        fs.copyFileSync(sourcePath, targetPath);
      });
    }
  } catch (error) {
    console.log(chalk.yellow('‚ö†Ô∏è Warning: Could not copy images from source directory'));
  }
}
  • Ensures images are copied to the correct directory for each theme.
  • Creates directories recursively, preventing errors for non-existent paths.
  • Uses a loop to handle multiple images, making it scalable.
copyQuestNavigator(outputDir: string): void {
  const templatesDir = path.join(this.sourceDir, 'templates');
  const cssSource = path.join(templatesDir, 'quest-navigator.css');
  const jsSource = path.join(templatesDir, 'quest-navigator.js');

  const assetsDir = path.join(outputDir, 'assets');
  fs.mkdirSync(assetsDir, { recursive: true });

  if (fs.existsSync(cssSource)) {
    fs.copyFileSync(cssSource, path.join(assetsDir, 'quest-navigator.css'));
  }
  if (fs.existsSync(jsSource)) {
    fs.copyFileSync(jsSource, path.join(assetsDir, 'quest-navigator.js'));
  }
}
  • Copies navigation-related assets like CSS and JavaScript files for single-theme adventures.
  • Ensures all required files are present in the output directory for seamless navigation.
  • Uses fs.mkdirSync to create directories as needed, ensuring robustness.
copyGlobalAssets(outputDir: string): void {
  try {
    const globalSharedDir = path.join(outputDir, 'assets', 'shared');
    fs.mkdirSync(globalSharedDir, { recursive: true });

    const globalImages = ['github-mark.svg', 'github-mark-white.svg'];
    globalImages.forEach(file => {
      const sourcePath = path.join(this.sourceDir, 'assets', 'shared', file);
      if (fs.existsSync(sourcePath)) {
        const targetPath = path.join(globalSharedDir, file);
        fs.copyFileSync(sourcePath, targetPath);
      }
    });
  } catch (error) {
    console.log(chalk.yellow('‚ö†Ô∏è Warning: Could not copy global assets'));
    throw error;
  }
}
  • Handles shared assets like icons and logos for multi-theme adventures.
  • Uses a loop to ensure all required files are copied, improving maintainability.
  • Provides error handling to alert developers of missing assets.
