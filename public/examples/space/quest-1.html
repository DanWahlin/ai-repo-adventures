File: packages/generator/src/cli/template-engine.ts

The TemplateEngine class powers the rendering of HTML pages by replacing placeholders in templates with dynamic content. This module ensures flexibility in generating pages with varying themes and data, while caching optimizes performance.

Highlights

  • renderPage combines base and content templates into complete HTML pages.
  • replacePlaceholders dynamically substitutes variables in templates for customization.
  • loadTemplate caches template files to improve rendering speed and reduce file system access.

Code

renderPage(contentTemplate: string, variables: TemplateVariables): string {
  const baseTemplate = this.loadTemplate('base-template.html');
  const contentHtml = this.loadTemplate(contentTemplate);
  
  const renderedContent = this.replacePlaceholders(contentHtml, variables);
  
  const pageVariables = {
    ...variables,
    CONTENT: renderedContent
  };
  
  return this.replacePlaceholders(baseTemplate, pageVariables);
}
  • This method dynamically constructs HTML pages by combining base and content templates.
  • Placeholder replacement ensures templates are populated with real data, enabling customization.
  • The use of caching reduces redundant file reads, improving performance.
  • Modular design simplifies the addition of new templates for future themes.
  • The separation of base and content templates allows consistent styling across pages.

copyGlobalAssets(outputDir: string): void {
  const globalAssetsDir = path.join(outputDir, 'assets');
  const targetImagesDir = path.join(globalAssetsDir, 'images');
  fs.mkdirSync(targetImagesDir, { recursive: true });

  const globalImages = ['github-mark.svg', 'github-mark-white.svg'];
  globalImages.forEach(file => {
    const sourcePath = path.join(this.sourceDir, 'assets', 'shared', file);
    if (fs.existsSync(sourcePath)) {
      const targetPath = path.join(globalAssetsDir, 'shared', file);
      fs.copyFileSync(sourcePath, targetPath);
    }
  });

  const homepageCSSSource = path.join(this.sourceDir, 'themes', 'homepage.css');
  const cssPath = path.join(outputDir, 'assets', 'homepage.css');
  fs.writeFileSync(cssPath, fs.readFileSync(homepageCSSSource, 'utf-8'));
}
  • This method ensures all shared assets are correctly copied for multi-theme adventures.
  • The use of fallback mechanisms guarantees essential files are available even if optional resources are missing.
  • Consolidating assets into a global directory simplifies access for multi-theme adventures.
  • Robust error handling prevents interruptions during asset copying.
  • The modular design allows easy extension to support additional asset types.
