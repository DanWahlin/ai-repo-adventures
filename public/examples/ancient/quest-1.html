File: packages/generator/src/cli/template-engine.ts

The TemplateEngine class is the cornerstone of the HTML generation process, responsible for loading templates and replacing placeholders with dynamic content. It ensures that the generated pages maintain a consistent structure while allowing for customization through variables.

Highlights

  • loadTemplate: Loads and caches template files from the filesystem, optimizing performance for repeated operations.
  • replacePlaceholders: Replaces placeholders in templates with actual values, enabling dynamic content rendering.
  • renderPage: Combines a base template with a content-specific template to generate complete HTML pages.

Code

private loadTemplate(templateName: string): string {
  if (this.templateCache.has(templateName)) {
    return this.templateCache.get(templateName)!;
  }

  const templatePath = path.join(this.templatesDir, templateName);
  
  try {
    const content = fs.readFileSync(templatePath, 'utf-8');
    this.templateCache.set(templateName, content);
    return content;
  } catch (error) {
    throw new Error(`Failed to load template ${templateName}: ${error}`);
  }
}
  • Implements a caching mechanism to avoid redundant file reads, improving performance.
  • Uses fs.readFileSync for synchronous file operations, ensuring templates are loaded before rendering.
  • Throws detailed errors for missing or inaccessible templates, aiding debugging.
private replacePlaceholders(template: string, variables: TemplateVariables): string {
  let result = template;
  
  for (const [key, value] of Object.entries(variables)) {
    const placeholder = `{{${key}}}`;
    result = result.replace(new RegExp(placeholder, 'g'), value || '');
  }
  
  return result;
}
  • Uses regular expressions to find and replace all instances of placeholders.
  • Ensures that undefined variables are replaced with empty strings, preventing rendering issues.
  • Supports dynamic content injection, making it adaptable for various use cases.
renderPage(contentTemplate: string, variables: TemplateVariables): string {
  const baseTemplate = this.loadTemplate('base-template.html');
  const contentHtml = this.loadTemplate(contentTemplate);
  
  const renderedContent = this.replacePlaceholders(contentHtml, variables);
  const pageVariables = {
    ...variables,
    CONTENT: renderedContent
  };
  
  return this.replacePlaceholders(baseTemplate, pageVariables);
}
  • Combines base and content templates to create complete HTML pages.
  • Leverages replacePlaceholders to inject both global and page-specific variables.
  • Facilitates modular design by separating layout (base template) from content (content template).

The AssetManager class handles the copying and organization of static assets, such as images and CSS files, ensuring that each generated theme has the necessary resources.

Highlights

  • copyImages: Copies theme-specific and shared images to the output directory.
  • copyQuestNavigator: Copies navigation-related assets to the appropriate directories for single-theme generation.
  • copyGlobalAssets: Manages global asset copying for multi-theme generation, including shared CSS and images.

copyImages(outputDir: string, isMultiTheme: boolean = false): void {
  const sourceImagesDir = path.join(this.sourceDir, 'assets', 'images');
  const targetImagesDir = path.join(outputDir, 'assets', 'images');

  try {
    if (fs.existsSync(sourceImagesDir)) {
      fs.mkdirSync(targetImagesDir, { recursive: true });
      const imageFiles = fs.readdirSync(sourceImagesDir);
      imageFiles.forEach(file => {
        const sourcePath = path.join(sourceImagesDir, file);
        const targetPath = path.join(targetImagesDir, file);
        fs.copyFileSync(sourcePath, targetPath);
      });
    }
  } catch (error) {
    console.log(chalk.yellow('‚ö†Ô∏è Warning: Could not copy images from source directory'));
  }
}
  • Ensures directories are created before copying files, preventing runtime errors.
  • Uses fs.existsSync to check for source directories, allowing for conditional execution.
  • Logs warnings for missing assets instead of throwing errors, ensuring the pipeline continues.
copyQuestNavigator(outputDir: string): void {
  const templatesDir = path.join(this.sourceDir, 'templates');
  const cssSource = path.join(templatesDir, 'quest-navigator.css');
  const jsSource = path.join(templatesDir, 'quest-navigator.js');
  const svgSource = path.join(templatesDir, 'compass-icon.svg');

  const assetsDir = path.join(outputDir, 'assets');
  fs.mkdirSync(assetsDir, { recursive: true });

  if (fs.existsSync(cssSource)) {
    fs.copyFileSync(cssSource, path.join(assetsDir, 'quest-navigator.css'));
  }
  if (fs.existsSync(jsSource)) {
    fs.copyFileSync(jsSource, path.join(assetsDir, 'quest-navigator.js'));
  }
  if (fs.existsSync(svgSource)) {
    fs.copyFileSync(svgSource, path.join(assetsDir, 'compass-icon.svg'));
  }
}
  • Copies navigation-related CSS, JavaScript, and SVG files to the output directory.
  • Ensures all required assets are available for the quest navigation system.
  • Uses fs.mkdirSync with recursive: true to handle nested directory creation.
