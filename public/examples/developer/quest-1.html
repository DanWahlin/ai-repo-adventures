File: packages/generator/src/cli/template-engine.ts

The TemplateEngine class handles template rendering and variable substitution, forming the backbone of HTML generation. It ensures that content placeholders are dynamically replaced with actual values, enabling the creation of complete, styled pages for adventures.

Highlights

  • loadTemplate caches and loads HTML templates from the filesystem for efficient reuse.
  • replacePlaceholders substitutes placeholders in templates with actual values provided in variables.
  • renderPage combines base and content templates to generate fully rendered pages.

Code

private loadTemplate(templateName: string): string {
  if (this.templateCache.has(templateName)) {
    return this.templateCache.get(templateName)!;
  }

  const templatePath = path.join(this.templatesDir, templateName);
  
  try {
    const content = fs.readFileSync(templatePath, 'utf-8');
    this.templateCache.set(templateName, content);
    return content;
  } catch (error) {
    throw new Error(`Failed to load template ${templateName}: ${error}`);
  }
}
  • Caches templates to avoid redundant file reads and improve performance.
  • Uses fs.readFileSync for synchronous file reading, ensuring template availability during rendering.
  • Throws meaningful errors to assist debugging when templates fail to load.
private replacePlaceholders(template: string, variables: TemplateVariables): string {
  let result = template;
  
  for (const [key, value] of Object.entries(variables)) {
    const placeholder = `{{${key}}}`;
    result = result.replace(new RegExp(placeholder, 'g'), value || '');
  }
  
  return result;
}
  • Dynamically replaces placeholders ({{key}}) in templates with corresponding variable values.
  • Uses regular expressions for global replacement, ensuring all instances of a placeholder are updated.
  • Supports fallback values ('') for missing variables, maintaining template integrity.
renderPage(contentTemplate: string, variables: TemplateVariables): string {
  const baseTemplate = this.loadTemplate('base-template.html');
  const contentHtml = this.loadTemplate(contentTemplate);
  
  const renderedContent = this.replacePlaceholders(contentHtml, variables);
  const pageVariables = { ...variables, CONTENT: renderedContent };
  
  return this.replacePlaceholders(baseTemplate, pageVariables);
}
  • Combines base and content templates to generate complete pages.
  • Ensures modularity by separating base layout from dynamic content.
  • Demonstrates the use of composition for scalable HTML generation.

File: packages/generator/src/cli/asset-manager.ts

The AssetManager class is responsible for copying and organizing assets like images, CSS, and shared files. It ensures that both theme-specific and global resources are properly placed for single and multi-theme adventures.

Highlights

  • copyImages handles theme-specific and shared image copying to output directories.
  • copyQuestNavigator organizes navigation assets for single-theme adventures.
  • copyGlobalAssets consolidates shared assets for multi-theme generation.

Code

copyImages(outputDir: string, isMultiTheme: boolean = false): void {
  const sourceImagesDir = path.join(this.sourceDir, 'assets', 'images');
  const targetImagesDir = path.join(outputDir, 'assets', 'images');

  try {
    if (fs.existsSync(sourceImagesDir)) {
      fs.mkdirSync(targetImagesDir, { recursive: true });
      const imageFiles = fs.readdirSync(sourceImagesDir);
      imageFiles.forEach(file => {
        const sourcePath = path.join(sourceImagesDir, file);
        const targetPath = path.join(targetImagesDir, file);
        fs.copyFileSync(sourcePath, targetPath);
      });
    }
  } catch (error) {
    console.log(chalk.yellow('‚ö†Ô∏è Warning: Could not copy images from source directory'));
  }
}
  • Ensures images are copied from source to target directories, creating required folders.
  • Uses fs.existsSync for conditional execution, avoiding errors when directories are missing.
  • Implements error handling to notify users of issues without crashing the pipeline.
copyQuestNavigator(outputDir: string): void {
  const templatesDir = path.join(this.sourceDir, 'templates');
  const cssSource = path.join(templatesDir, 'quest-navigator.css');
  const assetsDir = path.join(outputDir, 'assets');

  fs.mkdirSync(assetsDir, { recursive: true });

  if (fs.existsSync(cssSource)) {
    fs.copyFileSync(cssSource, path.join(assetsDir, 'quest-navigator.css'));
  }
}
  • Copies navigation-related assets like CSS to the appropriate output directory.
  • Modularizes asset management for single-theme adventures.
  • Ensures directory creation before file operations to prevent runtime errors.
