async initializeAdventure(projectInfo: ProjectInfo, theme: AdventureTheme, projectPath?: string, customThemeData?: CustomThemeData): Promise<string> {
    this.state.reset();
    this.state.projectInfo = projectInfo;
    this.state.currentTheme = theme;
    this.state.projectPath = projectPath || process.cwd();
    if (theme === 'custom' && customThemeData) {
      this.storyGenerator.setCustomTheme(customThemeData);
    }
    const storyResponse = await this.storyGenerator.generateStoryAndQuests(projectInfo, theme, this.state.projectPath);
    this.state.title = storyResponse.title;
    this.state.story = typeof storyResponse.story === 'string' ? storyResponse.story : storyResponse.story.content;
    this.state.quests = storyResponse.quests;
    return this.formatStoryWithQuests(storyResponse);
  }

The initializeAdventure function mirrors this orchestrated prelude by resetting existing data (clearing tables) and setting a theme (choosing the menu). It then syncs the project path (preps the kitchen), applies custom configurations—if provided (special dietary menus), and generates a list of quests (curated dishes). Finally, it composes a story narrative to accompany these quests (plating and presentation), ensuring a polished start to every spaced-out adventure session.

async exploreQuest(choice: string): Promise {
    const sanitizedChoice = this.validateAndSanitizeChoice(choice);
    if (this.isProgressRequest(sanitizedChoice)) {
      return this.getProgress();
    }
    const quest = this.findQuest(sanitizedChoice);
    if (!quest) {
      return this.createNotFoundResult();
    }
    return await this.executeQuest(quest);
  }

This function is akin to a librarian helping you locate a book in a vast library: it first validates your request (validateAndSanitizeChoice), checks if you’re asking for progress stats (isProgressRequest), and searches for the quest in the active repository (findQuest). If the quest isn’t found, it provides alternatives (createNotFoundResult). If located, it retrieves the full quest details via executeQuest, much like how the librarian finds the requested book, lays it out, and prepares it for reading.
